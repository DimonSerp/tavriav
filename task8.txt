Регламент обмена данными интернет­магазина сети продуктовых магазинов с внутренней учетной системой.

На сайт должна импортироваться информация о товарах, а экспортироваться данные о заказах.

1. Обмен данными построен на архитектуре RESTful, данные передаются в формате JSON
2. Точка "входа" https://<site.name>/api
3. Идентификация учетной системы (клиента для запросов) осуществляется с помощью токенов, которые передаются как параметр при каждом запросе.
3. Пример выгрузки на сайт товаров:
Учетная система делает запрос на URL ("https://<site.name>/api/products") c Request Method:PUT и данными товаров в теле запроса в формате JSON
3. Пример экспорта с сайта заказов:
Учетная система делает запрос на URL ("https://<site.name>/api/orders") c Request Method:GET и параметрами в запросе: период (даты) заказов
4. Запросы на выгрузки (импорт на сайт) товаров из внутренней системы учета могут быть по времени и по событию добавления/изменения товаров
5. Запросы на экспорт заказов с сайта осуществляется по времени, например 1 раз в 5 минут.
